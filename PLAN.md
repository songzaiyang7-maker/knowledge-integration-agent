## 四大核心问题（评委关注重点，必须想清楚）

### Q1：知识点的粒度如何定义？
**定义**：一个知识点 = 一个可独立考查的概念/定理/方法/现象

**操作规则**：
- 按"段落级"提取，对应教材约 1 段到 1 页的独立概念
- LLM Prompt 约束：每个知识点必须有**独立定义、可单独解释、可出考题**
- 如果两个概念总是一起出现且无法独立解释 → 合并为一个知识点
- **举例**：✅ "动作电位"（可独立考查） ❌ "静息电位和动作电位"（应拆两个） ❌ "生理学"（太粗）

### Q2：整合后如何保证教学连贯性？
**核心策略**：基于 prerequisite 链路的完整性检查

- 知识点间有 `prerequisite` 关系 → 形成教学链路
- **链路完整性原则**：
  - 保留 B → B 的所有前置依赖 A 也必须保留
  - 删除 A → 检查 A 的所有下游节点是否"断链"
- **连贯性审计**：整合后自动遍历所有 prerequisite 关系，标记"孤儿节点"（前置被删但自身保留）
- **决策优先级**：前置依赖多的"枢纽知识点"优先保留

### Q3："重复"的判定标准是什么？
**三级判定体系**：

| 级别 | 方法 | 标准 | 置信度 | 决策 |
|------|------|------|--------|------|
| L1 | 名称精确匹配 | 字符串一致 | ≥ 0.95 | merge |
| L2 | Embedding 语义相似度 | 余弦 > 0.85 | 0.85-0.95 | merge |
| L3 | LLM 语义精判 | MiniMax 判断等价 | 0.7-0.85 | flag（交教师审核） |
| — | 语义不相关 | < 0.7 | < 0.7 | keep（独立保留） |

- 跨语言等价（"白细胞" = "leukocyte"）由 L3 捕获
- 概念包含关系（"免疫应答" ⊃ "体液免疫"）不算重复，标记为 `contains` 关系

### Q4：30% 压缩比如何计算？
**公式**：压缩比 = 整合后知识点定义文本总字数 / 原始教材知识点定义文本总字数 × 100%

**三种压缩策略**：

| 策略 | 操作 | 贡献 |
|------|------|------|
| 去重合并 | 5 本书都讲"炎症" → 保留最完整版本 | ~40% |
| 精炼提纯 | LLM 提炼保留定义的精华表述 | ~30% |
| 冗余删除 | 完全重复的知识点只保留一个 | ~30% |

**前端展示**：原始 X 个知识点 X 字 → 整合后 Y 个知识点 Y 字 → 压缩比 Z%

## 五大出众策略

### 策略一：文档与架构的"降维打击"（35分 = A 15分 + D 20分）
**核心思路**：文档不是附属品，是主战场。很多人5小时全写代码，忽略了1/3的分数在这里。

- `docs/Agent架构说明.md` 是王牌文档，必须包含：
  - Mermaid 架构图（调用链路清晰可见）
  - 单 Agent vs 多 Agent 的设计决策论证（为什么不拆？时间约束下的上下文管理策略）
  - 已知局限与改进方向
  - **量化数据支撑**（Embedding 阈值对比、分块大小对比、Token 消耗统计）
- 用数据说话："Embedding 阈值 0.85 时对齐准确率最高"、"Chunk 500字 vs 800字的检索命中率差异"

### 策略二：跨教材图谱整合的"双重对齐"（核心技术亮点）
**核心思路**：这是赛题的"技术王冠"，单独占功能分 5+1=6 分。

- **三级对齐算法**（对应 Q3）：
  1. L1：名称精确匹配 → 直接 merge
  2. L2：BGE-small-zh 计算余弦相似度 > 0.85 → 候选池
  3. L3：MiniMax 2.7 对候选对做最终 merge/keep/remove 裁决
- **教学连贯性保障**（对应 Q2）：
  - 整合后自动执行 prerequisite 链路完整性审计
  - 枢纽知识点（前置依赖多）优先保留
  - 断链节点标记为"孤儿节点"，写入报告
- **压缩比精准控制**（对应 Q4）：
  - 三种策略：去重合并(~40%) + 精炼提纯(~30%) + 冗余删除(~30%)
  - 前端展示"合并前 vs 合并后"统计面板
  - 实时计算压缩比百分比（必须 < 30%）

### 策略三：RAG 的防幻觉与溯源（精准问答的严谨性）
**核心思路**：赛题强调"每一个回答都有据可查"。

- **元数据完美透传**：Chunking 阶段死死绑定"教材名、章节、页码"
- **严格 Prompt 约束**：系统 Prompt 中明确"如果上下文中没有，回答'当前知识库中未找到相关信息'，绝不允许编造"
- **UI 溯源体验**：引用来源可点击展开查看原文 chunk 内容（这个细节做出来体验分拉满）

### 策略四：视觉震撼力（13分 C 项）
**核心思路**：评委几分钟内评审，第一印象取决于界面。

- ECharts 力导向图，自带物理动效
- **视觉编码体现业务逻辑**：
  - 节点大小 = 知识点在多本教材中出现的频次
  - 节点颜色 = 所属教材来源
  - 点击节点 → 侧边栏弹出定义和出处
  - 整合前后对比视图

### 策略五：极端交付纪律
**核心思路**："完成度"本身就是最大的出众点。

- **坚决 Mock，保证闭环**：任何环节卡住，立刻用 JSON 备用方案绕过
- **最后 1 小时锁代码**：不再加功能，全部写文档 + 部署
- **部署 > 完美**：跑不起来 = 0 分，简陋但能跑 = 碾压 60% 的人

## Phase 推进计划

### Phase 1：基础设施 + Mock 数据（25 分钟）

   - gradio, pymupdf, sentence-transformers, faiss-cpu, openai, numpy
3. 编写 `src/config.py`：MiniMax 2.7 API 配置（endpoint + key）
4. 编写 `src/mock_data.py`（策略五保险）：
   - 2 本教材的模拟解析结果（chapters JSON）
   - 模拟知识图谱节点和边（至少 15 个节点、20 条边，4 种关系类型）
   - 模拟整合决策（merge/keep/remove，含 confidence 和 reason）
   - 模拟 RAG 问答结果（带完整 citations + source_chunks）
5. 编写 `app.py` 骨架：Gradio Tabs（上传/图谱/问答/对话/报告）
6. 更新 `.gitignore`（添加 `*.pdf`, `data/`）

### Phase 3：后端真实逻辑（100 分钟）
**目标**：替换 Mock 为真实逻辑，实现核心亮点

1. **PDF 解析**（15 min）：
   - PyMuPDF 逐页解析
   - 章节识别（正则 "第X章"）
   - 页眉页脚过滤
   - 输出对齐 mock_data JSON

2. **知识点提取**（20 min）：
   - MiniMax 2.7 API 调用，Prompt 要求 JSON 输出
   - 提取知识点 + 识别关系（4 种）
   - 防幻觉 Prompt 设计

3. **跨教材整合 - 双重对齐**（30 min，策略二核心）：
   - BGE-small-zh 计算节点向量
   - 余弦相似度 > 0.8 → 候选池（Embedding 粗筛）
   - 候选对 → MiniMax 2.7 做 merge/keep/remove（LLM 精判）
   - 压缩比控制（< 30%）
   - **记录阈值对比数据**（用于文档策略一）

4. **RAG Pipeline**（25 min，策略三核心）：
   - 分块（500 字，50 字重叠），元数据绑定（教材名/章节/页码）
   - BGE-small-zh 向量化 → FAISS 索引
   - Top-5 检索 + MiniMax 2.7 生成带引用回答
   - 防幻觉 Prompt（"没有则回答未找到"）
   - **记录分块大小对比数据**（用于文档策略一）

5. **多轮对话**（10 min）：
   - 教师反馈 → 修改决策 → 更新图谱

6. **验收**：上传 PDF → 解析 → 图谱 → 整合 → RAG 问答 → 对话修改
