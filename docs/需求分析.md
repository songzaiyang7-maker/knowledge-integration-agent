# 需求分析文档

## 一、项目背景

在医学教育中，一门学科往往有 5-7 本不同版本的教材并行使用（如生理学、病理学、组织学与胚胎学等）。这些教材在知识体系上存在大量重叠和差异，教师和学生在使用时面临以下核心痛点：

- **知识点重复率高**：同一概念（如"白细胞""免疫应答"）在多本教材中反复出现，表述不一
- **缺乏统一视角**：没有工具能将多本教材的知识体系整合为一个连贯的知识图谱
- **教学效率低**：教师需要手动比对不同教材，耗时长且容易遗漏

本项目旨在构建一个**学科知识整合智能体**，通过 AI 技术自动完成多教材的知识提取、图谱构建、跨教材整合和 RAG 问答。

### 数据支撑

- 赛题要求处理的教材：**7 本**（生理学、病理学、组织学与胚胎学、医学微生物学、局部解剖学、传染病学、病理生理学）
- 典型医学教材规模：每本 **400-600 页**，含 **15-25 章**，总字数约 **30-50 万字**
- 预估跨教材知识点重叠率：**40-60%**（基于医学教材同质化程度）

---

## 二、四大核心问题

### Q1：知识点的粒度如何定义？

**定义**：一个知识点 = 一个可独立考查的概念/定理/方法/现象

**操作规则**：
- 按"段落级"提取，对应教材约 1 段到 1 页的独立概念
- LLM Prompt 约束：每个知识点必须有**独立定义、可单独解释、可出考题**
- 如果两个概念总是一起出现且无法独立解释 → 合并为一个知识点

**举例**：
- ✅ "动作电位"（可独立考查）
- ❌ "静息电位和动作电位"（应拆两个）
- ❌ "生理学"（太粗，不可独立考查）

**数据支撑**：
- 典型医学教材单章可提取 **15-40 个**独立知识点
- 7 本教材预估总知识点 **200-350 个**（整合前），整合后预计压缩至 **60-100 个**
- 粒度标准参照医学考试出题单位：一个知识点对应 **1 道选择题**或 **1 个名词解释**

---

### Q2：整合后如何保证教学连贯性？

**核心策略**：基于 prerequisite 链路的完整性检查

- 知识点间有 `prerequisite` 关系 → 形成教学链路
- **链路完整性原则**：
  - 保留 B → B 的所有前置依赖 A 也必须保留
  - 删除 A → 检查 A 的所有下游节点是否"断链"
- **连贯性审计**：整合后自动遍历所有 prerequisite 关系，标记"孤儿节点"（前置被删但自身保留）
- **决策优先级**：前置依赖多的"枢纽知识点"优先保留

**数据支撑**：
- 医学教材中 prerequisite 关系占比约 **25-35%**（知识体系呈明显的层级依赖结构）
- 枢纽知识点（被 ≥3 个下游节点依赖）占比约 **10-15%**，这些是整合时必须保留的核心节点
- 整合后通过 prerequisite 链路审计，"孤儿节点"应控制在 **0 个**

---

### Q3："重复"的判定标准是什么？

**三级判定体系**：

| 级别 | 方法 | 标准 | 置信度 | 决策 |
|------|------|------|--------|------|
| L1 | 名称精确匹配 | 字符串一致 | ≥ 0.95 | merge |
| L2 | Embedding 语义相似度 | 余弦 > 0.85 | 0.85-0.95 | merge |
| L3 | LLM 语义精判 | MiniMax 判断等价 | 0.7-0.85 | flag（交教师审核） |
| — | 语义不相关 | < 0.7 | < 0.7 | keep（独立保留） |

**补充规则**：
- 跨语言等价（"白细胞" = "leukocyte"）由 L3 捕获
- 概念包含关系（"免疫应答" ⊃ "体液免疫"）不算重复，标记为 `contains` 关系

**数据支撑**：
- 7 本教材预估知识点配对数：C(300, 2) = **44,850 对**（需逐一判断）
- L1 精确匹配预估命中：**10-15%**（如"炎症""红细胞"等标准化术语）
- L2 语义相似预估命中：**20-30%**（如"T细胞" vs "T淋巴细胞"）
- L3 LLM 精判预估触发：**5-10%**（需要高级语义理解的边界情况）
- 最终需教师审核（flag）的决策预估 **5-10 项**

---

### Q4：30% 压缩比如何计算？

**公式**：

```
压缩比 = 整合后知识点定义文本总字数 / 原始教材知识点定义文本总字数 × 100%
```

**三种压缩策略**：

| 策略 | 操作 | 贡献 |
|------|------|------|
| 去重合并 | 5 本书都讲"炎症" → 保留最完整版本 | ~40% |
| 精炼提纯 | LLM 提炼保留定义的精华表述 | ~30% |
| 冗余删除 | 完全重复的知识点只保留一个 | ~30% |

**前端展示**：原始 X 个知识点 X 字 → 整合后 Y 个知识点 Y 字 → 压缩比 Z%

**数据支撑**：
- 7 本教材原始知识点定义预估总字数：**150,000-250,000 字**
- 目标压缩后：**45,000-75,000 字**（30%）
- Mock 数据验证（2 本教材）：18 个节点 / 745,000 字 → 14 个节点 / 195,000 字 = **26.2%**（达标）

## 三、非功能需求

| 维度 | 要求 |
|------|------|
| 响应时间 | 整合操作 < 5 分钟（7 本教材），RAG 单次问答 < 10 秒 |
| 可视化 | 知识图谱支持缩放、拖拽、节点点击详情 |
| 可解释性 | 每项整合决策附带理由和置信度，RAG 回答附带引用来源 |
| 部署 | 支持魔搭创空间一键部署（Gradio 原生支持） |
